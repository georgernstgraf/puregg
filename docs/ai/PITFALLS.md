# Pitfalls

Things that do not work, subtle bugs, and non-obvious constraints.
Read this file carefully before making changes in affected areas.

- Decompiled APK resource collisions with support libraries: Must manually remove duplicate `<attr>` definitions in `attrs.xml`.
- `BuildConfig.java` is auto-generated by Gradle and must be removed if it was included in the decompiled output.
- `if (context == 0)` in decompiled Java code: Must be manually changed to `if (context == null)`.
- Resource and asset merge failures: Decompiled library-internal resources (`abc_*`, `design_*`) conflict with `appcompat` and `material` dependencies. These MUST be removed from `app/src/main/res/` to allow Gradle to resolve them correctly.
- AndroidX migration: Incompatible legacy resource references in `styles.xml` (like `android:backgroundColor` instead of `android:background` or referencing non-existent library drawables) cause build failures.
- Java 17 compatibility: Legacy code might need explicit `androidx` migrations for `Fragment` and `FragmentManager`.
- CI/CD build issues: Instrumented tests require a functioning emulator and clean resource environment, which is currently blocked by resource linking errors.
